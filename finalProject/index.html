CS184 Final Project

Pasta Factory

In this project, we will be simulating the physical interaction of cutting pasta from dough. We can then further physically interact with this pasta using the GUI.

Team: Brian Quach, Jeffrey Huang, Jeremy Li, Emily Xiao

Problem Description

We want to build a virtual interface to make pasta cutting accessible to all - it isn’t important per se, but will be fun to use in lieu of real dough.

Where it is challenging: We plan to make the simulation interactive, meaning you can physically move the objects within the scene.

Goals and Deliverables

We want the simulation to run in realtime. Some questions that we hope to answer include:

  • How will we correctly divide the mesh and then efficiently maintain data structures to track separate meshes?
  • What happens when the pasta is too long. Will it break, and if so, how?
  • How will we simulate the friction force applied between strands of pasta?
  • How will the noodle fold on itself as it falls into the bowl?

We plan to create a demo where a sheet of dough is processed through a slicer, dividing the dough into many individual noodles. These noodles will then fall into a bowl, where they will be penetrated with a fork and then stirred by the fork.

We hope to add an additional conveyor belt so that the pasta falls not directly into the bowl but onto a belt where it reaches another stage of the pasta process: Cooking, which would change the texture of the pasta. Then it goes into the bowl.

Schedule

  • Week 1:
  • Set up the simulation environment, and create the deformable sheet dough and cutter. Enable dough mesh divisibility.
  • Week 2:
  • Enable the dough to be split into separate entities and run the dough through the slicer
  • Week 3:
  • Implement collision between the fork and the noodles
  • Week 4:
  • Apply force onto the noodles and the force interaction between separate noodles
